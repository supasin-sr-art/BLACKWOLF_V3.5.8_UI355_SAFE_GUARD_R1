Quick Guide - Master + เช็คสถานะ ISSUE พร้อมไว้ข้ามวัน

LOCAL PROCESSING
1

แนบไฟล์หลัก 4 ไฟล์

Master, เช็คสถานะ ISSUE, Daily Report และ M190

แนบ SM / Blacklist / Auto-Mail 7.2 ตามที่มี

2

ระบบจำแนกจากโครงสร้าง

ตรวจชื่อ Sheet และ Header เป็นหลัก

ชื่อไฟล์ใช้ช่วยเฉพาะกรณี SM / Blacklist ที่โครงสร้างคล้ายกัน

3

ตรวจ Date คอลัมน์ T

อ่านวันเริ่มต้นจาก Master > Data > Date

กรอก Manual Start Date เฉพาะเมื่อ Master ไม่มี Date ที่อ่านได้

4

สร้าง Check และ ETL ใหม่

Check = M190 รอบปัจจุบัน

ETL = Auto-Mail 7.2 รอบปัจจุบัน ไม่เหลือข้อมูลเก่าค้าง

5

Carry Forward รายการค้าง

เก็บ Backlog, SM และ Blacklist เดิมไว้

รวมรายการเดิมด้วย CertificateNo หรือ alienCode + ProposalID ก่อนตัดรายการที่ออกกรมธรรม์แล้ว

6

ดาวน์โหลดไฟล์หลัก 2 ไฟล์

Master มีสูตร P:W และ ISSUE มี Data/Check/ETL ใหม่

ชื่อไฟล์มีวันที่ Run และต้องดาวน์โหลดเก็บเองทุกวัน

หมายเหตุ: Run History เก็บใน Browser ชั่วคราว 4 วัน และอาจหายเมื่อ Clear Browser Data ดาวน์โหลด Master + ISSUE หลัง Run ทุกครั้ง
